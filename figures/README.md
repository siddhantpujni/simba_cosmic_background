# Figures

This directory contains generated plots, visualizations, and figures for papers and presentations.

## Purpose

Store all generated figures here:
- Exploratory plots from analysis
- Publication-quality figures
- Presentation slides and graphics
- Diagnostic plots

## Organization

Organize by purpose or paper section:

```
figures/
├── exploratory/           # Initial exploration plots
├── paper/                # Final publication figures
│   ├── figure1_schematic.pdf
│   ├── figure2_sed_comparison.pdf
│   └── figure3_background_evolution.pdf
├── presentations/        # Talk slides and graphics
└── diagnostics/         # QA and diagnostic plots
```

Or use flat structure with descriptive names:
```
figures/
├── 2024-01-15_galaxy_distribution.png
├── paper_fig1_schematic.pdf
├── paper_fig2_sed_comparison.pdf
└── talk_2024-02_main_results.pdf
```

## File Formats

### For Papers
- **Vector formats**: PDF, EPS (preferred for publications)
- **Raster formats**: PNG (300+ dpi) for complex plots
- Avoid JPEG for scientific figures (lossy compression)

### For Presentations
- PNG (high resolution)
- PDF (for LaTeX beamer)

### For Web/Documentation
- PNG or SVG

## Naming Convention

Use descriptive, version-controlled names:

```
[date]_[description]_[version].ext
2024-01-15_sed_comparison_v1.pdf
2024-01-20_sed_comparison_v2.pdf

paper_fig[number]_[description].pdf
paper_fig1_schematic.pdf
paper_fig2_data_comparison.pdf
```

## Matplotlib Settings

For consistent, publication-quality figures, use a style file:

```python
import matplotlib.pyplot as plt

# Set publication defaults
plt.style.use('seaborn-paper')
plt.rcParams.update({
    'font.size': 12,
    'axes.labelsize': 14,
    'axes.titlesize': 14,
    'xtick.labelsize': 12,
    'ytick.labelsize': 12,
    'legend.fontsize': 10,
    'figure.figsize': (8, 6),
    'figure.dpi': 100,
    'savefig.dpi': 300,
    'savefig.bbox': 'tight',
    'savefig.format': 'pdf'
})
```

Or create a style file in `src/styles/publication.mplstyle`.

## Best Practices

1. **High resolution**: Save at 300+ dpi for publications
2. **Vector when possible**: Use PDF/EPS for line plots and diagrams
3. **Descriptive filenames**: Include date and content description
4. **Version control**: Keep previous versions when iterating
5. **Accessibility**: Use colorblind-friendly palettes
6. **Labels**: Always include axis labels with units
7. **Legends**: Add clear legends or annotations
8. **File size**: Optimize for reasonable file sizes (< 5MB per figure)

## Color Palettes

Recommended colorblind-friendly palettes:
- Seaborn: `colorblind`, `muted`
- ColorBrewer: `Set2`, `Dark2`
- Scientific: `viridis`, `plasma`, `cividis`

## Generating Figures

Figures should be generated by scripts or notebooks:

```python
# In scripts/generate_paper_figures.py or notebooks/
import matplotlib.pyplot as plt

fig, ax = plt.subplots(figsize=(8, 6))
# ... plotting code ...
plt.savefig('figures/paper/figure1_schematic.pdf', dpi=300, bbox_inches='tight')
plt.close()
```

## Figure Captions

Keep a separate file with figure captions for the paper:

```
figures/captions.txt or figures/captions.md
```

This helps when writing the paper and ensures consistency.

## Git Tracking

- Track final publication figures in git
- Consider `.gitignore` for large exploratory plots
- Use Git LFS if figures are very large
